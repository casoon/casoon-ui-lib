Grundlagen

Die Grundlagen der Casoon UI Library umfassen Design-Tokens, Farben, Typografie, Abstände und mehr.

Design-Tokens

Design-Tokens sind die grundlegenden Bausteine unseres Designsystems. Sie definieren konsistente Werte für Farben, Typografie, Abstände und mehr.

Best Practices

Zugänglichkeit

<button 
    class="button"
    aria-label="Schließen"
    aria-expanded="false"
    aria-controls="menu"
>
    Menü
</button>

Performance

.element {
    will-change: transform;
    transform: translateZ(0);
    backface-visibility: hidden;
    perspective: 1000px;
}

Wartbarkeit

/* Komponenten */
.component {
    /* Basis-Styles */
}

/* Varianten */
.component--variant {
    /* Varianten-Styles */
}

/* Zustände */
.component.is-active {
    /* Zustands-Styles */
}

Responsive Design

Unsere Komponenten sind für alle Bildschirmgrößen optimiert:

Mobile First

/* Basis-Styles für mobile */
.component {
    padding: var(--spacing-md);
}

/* Anpassungen für größere Bildschirme */
@media (min-width: var(--breakpoint-md)) {
    .component {
        padding: var(--spacing-lg);
    }
}

Fluide Typografie

.text {
    font-size: clamp(
        var(--font-size-sm),
        1vw + 1rem,
        var(--font-size-lg)
    );
}

Adaptive Layouts

.grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
    gap: var(--spacing-md);
}

Entwicklung

Richtlinien für die Entwicklung mit der Casoon UI Library:

Installation

npm install casoon-ui-lib

Verwendung

@import 'casoon-ui-lib/core.css';

Customization

:root {
    --color-primary: #007bff;
    --color-secondary: #6c757d;
}

Build

npm run build

Test

npm run test

Lint

npm run lint
