Logische Properties für Internationalisierung

Mit logischen Properties können wir Layouts erstellen, die unabhängig von der Schreibrichtung funktionieren. Dies ist besonders wichtig für mehrsprachige Webseiten, die sowohl LTR (left-to-right) als auch RTL (right-to-left) Sprachen unterstützen müssen.

Was sind logische Properties?

Logische Properties basieren auf der Flussrichtung des Dokuments anstatt auf physischen Richtungen wie links, rechts, oben oder unten. Die Hauptbegriffe sind:

  • Inline: Die Richtung, in der Text fließt (horizontal in vielen Sprachen)
  • Block: Die Richtung, in der Blöcke gestapelt werden (vertikal in vielen Sprachen)
  • Start: Der Anfang in der jeweiligen Richtung (links in LTR, rechts in RTL)
  • End: Das Ende in der jeweiligen Richtung (rechts in LTR, links in RTL)

Vergleich: Physische vs. Logische Properties

Beispiel für Margins und Paddings

Physische Properties

.element { margin-left: 1rem; margin-right: 2rem; padding-left: 1.5rem; padding-right: 1.5rem; border-left: 2px solid blue; } /* Benötigt extra Regeln für RTL */ [dir="rtl"] .element { margin-left: 2rem; margin-right: 1rem; border-left: none; border-right: 2px solid blue; }
Dieser Box verwendet physische Properties und benötigt spezielle Anpassungen für RTL.

Logische Properties

.element { margin-inline-start: 1rem; margin-inline-end: 2rem; padding-inline: 1.5rem; border-inline-start: 2px solid blue; } /* Keine zusätzlichen Regeln für RTL nötig! */
Diese Box verwendet logische Properties und passt sich automatisch an die Schreibrichtung an.

Text-Ausrichtung

Physische Ausrichtung

.element { text-align: left; } [dir="rtl"] .element { text-align: right; }

Dieser Text ist physisch links ausgerichtet und erfordert besondere Regeln für RTL-Sprachen.

Logische Ausrichtung

.element { text-align: start; }

Dieser Text ist logisch am Start ausgerichtet und passt sich automatisch an die Schreibrichtung an.

Allgemeine Anwendungsfälle

Flex Layouts

Flex Layouts profitieren von logischen Properties, besonders wenn die Flexrichtung von der Schreibrichtung abhängt.

.container { display: flex; justify-content: space-between; gap: var(--spacing-base); } .item { padding-inline: 1rem; margin-inline-end: 0.5rem; border-inline-start: 2px solid var(--color-primary); }

Grid Layouts

Grid Layouts können auch von logischen Properties profitieren, besonders bei der Positionierung von Elementen.

.container { display: grid; grid-template-columns: repeat(3, 1fr); gap: var(--spacing-base); } .item { padding-block: 1rem; border-block-start: 2px solid var(--color-secondary); text-align: center; }

Absolute Positionierung

Logische Properties sind besonders nützlich bei absoluter Positionierung.

.element { position: absolute; inset-block-start: 20px; /* statt top */ inset-inline-end: 20px; /* statt right in LTR, left in RTL */ }

Alle logischen CSS-Properties im Überblick

Margin

  • margin-block-start (statt margin-top)
  • margin-block-end (statt margin-bottom)
  • margin-inline-start (statt margin-left in LTR)
  • margin-inline-end (statt margin-right in LTR)
  • margin-block (Shorthand für margin-block-start und margin-block-end)
  • margin-inline (Shorthand für margin-inline-start und margin-inline-end)

Padding

  • padding-block-start (statt padding-top)
  • padding-block-end (statt padding-bottom)
  • padding-inline-start (statt padding-left in LTR)
  • padding-inline-end (statt padding-right in LTR)
  • padding-block (Shorthand für padding-block-start und padding-block-end)
  • padding-inline (Shorthand für padding-inline-start und padding-inline-end)

Border

  • border-block-start (statt border-top)
  • border-block-end (statt border-bottom)
  • border-inline-start (statt border-left in LTR)
  • border-inline-end (statt border-right in LTR)
  • border-block, border-inline (Shorthands)
  • Auch verfügbar: border-*-width, border-*-style, border-*-color

Size

  • inline-size (statt width)
  • block-size (statt height)
  • min-inline-size, max-inline-size
  • min-block-size, max-block-size

Positionierung

  • inset-block-start (statt top)
  • inset-block-end (statt bottom)
  • inset-inline-start (statt left in LTR)
  • inset-inline-end (statt right in LTR)
  • inset-block, inset-inline (Shorthands)
  • inset (Super-Shorthand für alle Seiten)

Overflows

  • overflow-inline (Überlauf in Textrichtung)
  • overflow-block (Überlauf in Blockrichtung)

Text-Ausrichtung

  • text-align: start (statt left in LTR)
  • text-align: end (statt right in LTR)

Implementierungsstrategie

Für eine internationalisierte Webseite empfehlen wir:

  1. Verwenden Sie durchgängig logische Properties statt physischer Properties
  2. Setzen Sie das dir Attribut auf rtl für RTL-Sprachen
  3. Testen Sie Ihr Layout in beiden Richtungen
  4. Beachten Sie, dass manche Elemente wie Icons möglicherweise gespiegelt werden müssen

Logische Properties werden von allen modernen Browsern unterstützt. Casoon UI verwendet konsequent logische Properties für optimale Internationalisierungsunterstützung.
